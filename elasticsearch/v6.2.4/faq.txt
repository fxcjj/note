原因：es不能以root用户运行

解决：创建新用户和组
# 创建组
groupadd es-normal
# 创建新用户并加入到es-normal组中，用户密码为123456
useradd puppet -g es-normal -p 123456
# 进入es所在目录，执行chown命令
chown -R puppet:es-normal elasticsearch-6.2.4
# 切换puppet到用户
su puppet
cd elasticsearch-6.2.4/bin
# 启动
./elasticsearch

# 修改用户密码
passwd puppet

3. max file descriptors [4096] for elasticsearch process is too low, increase to at least [65536]

问题翻译过来就是：elasticsearch用户拥有的可创建文件描述的权限太低，至少需要65536；

解决办法：

# 切换到root用户修改
vi /etc/security/limits.conf
 
# 在最后面追加下面内容，puppet表示启动ES的用户
puppet hard nofile 65536
puppet soft nofile 65536
 
4. max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]
问题翻译过来就是：elasticsearch用户拥有的内存权限太小，至少需要262144；

解决：

# 切换到root用户执行命令：
sysctl -w vm.max_map_count=262144

查看结果：
sysctl -a|grep vm.max_map_count

显示：
vm.max_map_count = 262144


上述方法修改之后，如果重启虚拟机将失效，所以：

解决办法：

在/etc/sysctl.conf文件最后添加一行

vm.max_map_count=262144

即可永久修改
